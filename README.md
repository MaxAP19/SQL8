# SQL8
Using of specific OLAP constructions to get a "Value" from data.

## Описание:
Проект посвящён работе с агрегированными данными и аналитическими SQL-запросами. Основной фокус — использование OLAP-подходов для получения бизнес-метрик и аналитической информации на основе транзакционных данных (OLTP).

В рамках проекта отрабатываются ключевые приёмы агрегации, группировки и анализа данных, позволяющие переходить от «сырых» событий (визитов и заказов) к обобщённым показателям, пригодным для принятия управленческих решений.

## Этапы выполнения проекта:
- Получение базовой агрегированной информации с использованием COUNT, AVG, MIN, MAX.
- Анализ активности пользователей и популярности ресторанов.
- Использование GROUP BY, HAVING и сортировки агрегированных данных.
- Работа с уникальностью данных без использования стандартных конструкций группировки.
- Объединение статистики визитов и заказов для расчёта итоговых метрик.
- Расчёт бизнес-показателей по ресторанам (количество заказов, средняя, минимальная и максимальная цена).
- Работа с глобальными агрегатами и округлением чисел.
- Анализ данных в разрезе адресов и локаций.
- Использование формул и явного преобразования типов в агрегированных вычислениях.

## Освоенные компетенции:
- Понимание различий между OLTP и OLAP-нагрузкой.
- Навыки построения аналитических SQL-запросов.
- Уверенное использование агрегатных функций и группировок.
- Применение HAVING для фильтрации агрегированных данных.
- Расчёт комплексных бизнес-метрик на основе нескольких источников данных.
- Работа с числовыми формулами, округлением и логическими выражениями в SQL.
- Подготовка данных, пригодных для аналитики и отчётности.
